## 調査結果サマリー

SLES SP5アップグレード前に確認すべき6項目を網羅した調査コマンドセット。
ディスク容量・サービス状態・ネットワーク設定・リポジトリ状態・インストール済みパッケージを収集し、
アップグレード可否を判断するための情報を取得する。

---

## 手順

### 1. OS情報の確認

```bash
uname -a
cat /etc/os-release
hostname
```

**確認ポイント:**
- OS が SLES 15 SP4 以前であること
- カーネルバージョンを記録
- ホスト名を記録

---

### 2. リポジトリ設定の確認

```bash
zypper lr
```

**確認ポイント:**
- SP5用リポジトリが未設定であること（アップグレード前は旧リポジトリのみ）
- 無効化済みリポジトリがないこと
- Enabled列が正しく設定されていること

---

### 3. カーネルパッケージ情報の確認

```bash
zypper info kernel-default
```

**確認ポイント:**
- 現在インストール済みのカーネルバージョン
- Available/Installed の状態

---

### 4. インストール済みパッケージ一覧の取得

```bash
rpm -qa
```

**確認ポイント:**
- パッケージ一覧をファイルに保存（アップグレード前後の差分確認用）
- サードパーティパッケージの有無

---

### 5. サービス状態の確認

```bash
systemctl status
```

**確認ポイント:**
- `Failed:` が 0 であること
- 異常終了しているサービスがないこと
- アップグレード対象サービスが起動中であること

---

### 6. ディスク使用状況の確認

```bash
df -h
```

**確認ポイント:**
- `/` (ルート) の空き容量が 10GB 以上あること
- `/boot` の空き容量が 200MB 以上あること
- 使用率が 80% を超えているパーティションがないこと

---

### 7. ネットワーク状態の確認

```bash
ip a
```

**確認ポイント:**
- 管理インターフェースが UP 状態であること
- IPアドレスが正しく割り当てられていること

---

### 8. 環境変数の確認

```bash
env
```

**確認ポイント:**
- プロキシ設定 (http_proxy / https_proxy) の有無
- PATH が正しく設定されていること

---

## 注意事項

1. **コマンドエラーについて:**
   - `zypper lr` / `rpm -qa` / `zypper info` はSLES環境でのみ動作する。Ubuntu/Debian環境では `command not found` となる。

2. **WSL2環境での制限:**
   - WSL2環境ではカーネルが `microsoft-standard-WSL2` カーネルとなり、通常のSLESカーネルとは異なる。

3. **アップグレード前必須確認事項:**
   - `systemctl status` で Failed ユニットが 0 であること
   - ディスク空き容量が十分であること（/ に最低 10GB）
   - リポジトリ設定が正しいこと

4. **バックアップ推奨:**
   - アップグレード前に必ず `/etc` のバックアップを取得すること（本スクリプトは読み取りのみ）
